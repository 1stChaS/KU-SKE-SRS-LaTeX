\chapter{Requirement Analysis}
\label{chap:requirement-analysis}

Requirement analysis is a critical phase in software development that ensures a clear understanding of what the system is expected to achieve. By identifying and refining requirements early in the development process, the project team can minimize risks, reduce implementation errors, and improve overall system quality. This phase focuses on understanding user needs, defining system functionalities, and establishing how different components of the system will interact. A well-defined requirement analysis helps ensure that the final system aligns with user expectations and project objectives.

In this chapter, we present the requirement analysis process for the AI-based UAV training system. The analysis includes stakeholder identification, user stories, use case diagrams, and user interface design. These components collectively define how the system operates and interacts with users, ensuring a practical, efficient, and user-centered design.


\section{Stakeholder Analysis}
\label{section:stakeholder-analysis}

Stakeholder analysis is an essential step in requirement analysis, as it identifies the key individuals or groups who interact with or are affected by the system. Understanding stakeholders’ needs and expectations ensures that the system is designed to meet real-world requirements and deliver meaningful value.
For this project, the primary stakeholder is the beginner drone pilot, who serves as the main user of the system. Beginner drone pilots are individuals with little or no prior experience in UAV flight planning and operation. These users require a structured, safe, and interactive learning environment to develop fundamental skills without the risks associated with real-world drone usage.

The system is designed to support beginner users by providing mission planning tools through QGroundControl, realistic flight simulation using ArduPilot SITL, and an engaging visualization interface developed in Unity. In addition, the system integrates AI-based evaluation to analyze user performance and provide feedback.

Furthermore, gamification elements such as adaptive training scenarios and a difficulty-normalized leaderboard are introduced to enhance user motivation and engagement. These features ensure that users can learn progressively while being fairly evaluated based on their skill level and training conditions.
By focusing on the needs of beginner drone pilots, the system aims to deliver an accessible and effective training platform that promotes continuous learning and skill improvement.

\section{User Stories and Use Cases}
\label{section:user-stories-use-cases}
User stories play an important role in defining system requirements from the perspective of end users. Instead of focusing on technical implementation, user stories describe what users need and why those needs are important. This approach ensures that the system development remains aligned with real-world usage and user expectations.

In this project, user stories are developed based on the identified target users, including beginner drone pilots, students and researchers, and training institutions. These user stories define the core functionalities of the system, such as mission planning, simulation, AI-based evaluation, and performance monitoring.

\begin{enumerate}
    \item As a beginner drone pilot, I want to learn how to plan a drone flight by setting waypoints in QGroundControl, so that I can understand flight planning concepts.

    \item As a beginner drone pilot, I want to visualize my drone flight in a 3D environment, so that I can better understand drone behavior.
    
    \item As a beginner drone pilot, I want to simulate my planned flight using ArduPilot SITL, so that I can test my flight plan in a safe environment.
    
    \item As a beginner drone pilot, I want the system to provide training scenarios with different difficulty levels, so that I can practice progressively based on my skill level.

    \item As a beginner drone pilot, I want to receive feedback after each mission, so that I can improve my flight planning skills.

    \item As a beginner drone pilot, I want to view my performance scores on a leaderboard, so that I can track my progress and stay motivated.

\end{enumerate}

Use case diagram (Figure~\ref{fig:use-case-diagram}) illustrates the interactions between users and the proposed AI-driven UAV training system. The breakdown of the diagram is as follows:

\noindent \textbf{Actors:}
\begin{itemize}
    \item \textbf{Beginner Drone Pilot:} The primary user of the system. Beginner drone pilots use the system to learn how to plan flight missions, simulate and visualize drone behavior, receive performance feedback, and practice through training scenarios with different difficulty levels to progressively improve their skills.
\end{itemize}

\noindent \textbf{Use Cases:}
\begin{itemize}
    \item \textbf{Plan Drone Flight:} The user creates a drone flight plan by defining mission parameters such as waypoints and flight paths. This allows the user to understand the fundamental concepts of UAV flight planning.
    \item \textbf{Simulate Flight:} The system executes the planned mission in a simulated environment using ArduPilot SITL. This allows the user to test and validate their flight plan safely without requiring real UAV hardware.
    \item \textbf{Visualize Flight in 3D:} The system provides a 3D visualization of the simulated flight, enabling the user to observe drone behavior such as movement, trajectory, and environmental interaction.
    \item \textbf{Receive Feedback:} After each simulation, the system evaluates the user's flight performance and provides feedback to help improve planning skills.
    \item \textbf{View Leaderboard:} The user can view their performance ranking compared to other users. This feature motivates users to improve their skills and track their progress over time.
    \item \textbf{Select Training Scenario (Difficulty Levels):} The user selects or is assigned training scenarios with varying levels of difficulty. This enables progressive learning by matching training complexity with the user’s skill level.
    
\end{itemize}

\label{section:user-interface-design}

The platform mainly uses green accents to represent progress, success, and system feedback, which aligns with the concept of skill development and performance improvement. \noindent 1. Homepage:
\begin{itemize}
    \item Serves as the main entry point of the system.
    \item Provides three primary options: Play, How to Play, and Settings.
    \item The Play button allows users to continue their training missions.
    \item The How to Play option provides guidance for beginners to understand the system and flight planning application (QGroundControl).
    \item The Settings option allows users to configure application preferences.
\end{itemize}

\noindent 2. Mission Selection Page:
\begin{itemize}
    \item Allows users to select available training missions based on their progress.
    \item Displays mission cards with mission title, description, and difficulty level.
    \item Shows locked missions that require completion of previous tasks.
    \item Provides user performance summary including number of missions completed, badges earned, and average score.
    \item Includes a performance panel displaying detailed metrics such as efficiency, safety, and completion rate.
\end{itemize}

\noindent 3. Instruction (How to Play) Page:
\begin{itemize}
    \item Provides step-by-step instructions for new users to start training.
    \item Includes guidance on downloading and setting up QGroundControl.
    \item Explains how to launch and connect to the simulator.
    \item Guides users on how to plan missions by creating waypoints and defining flight paths.
    \item Uses structured steps to improve clarity and ease of understanding.
\end{itemize}

\noindent 4. Settings Page:
\begin{itemize}
    \item Allows users to customize their training experience.
    \item Provides toggle options such as sound effects, fullscreen mode, and notifications.
    \item Includes a reset function to clear all mission progress and restart training.
    \item Uses simple controls and clear labels to ensure ease of interaction.
\end{itemize}

\noindent 5. Simulation Interface (Unity):
\begin{itemize}
    \item Provides a 3D environment for visualizing UAV flight behavior.
    \item Displays the drone, terrain, and camera perspective in real-time.
    \item Supports camera-follow functionality to track drone movement.
    \item Integrates with MAVLink communication for real-time simulation data.
    \item Enables users to observe flight performance safely without real-world risks.
\end{itemize}
